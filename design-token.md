## Token Architecture

Two-tier system:
- **Primitives** — raw values only, no intent encoded
- **Semantic** — intent only, aliases to primitives, no raw values

Theme support: Light / Dark (separate semantic sets)
Responsive support: Desktop / Mobile (typography only)

Flow: Primitive → Semantic → Component
Components never reference primitives directly.

## CSS Custom Property Convention

Pattern: `--{category}-{subcategory}-{role}`

```css
--color-surface-default
--color-text-primary
--color-interactive-primary
--color-border-focus
--spacing-md
--radius-card
--elevation-sm
--typography-body-md-fontSize
--typography-label-md-fontWeight
```

Components consume CSS custom properties only.
No raw hex, px, or hardcoded values in any component file.

## Rules DesignPull Enforces

1. Semantic tokens never contain raw values — aliases only
2. Components never reference primitive tokens directly
3. Light and dark semantic sets must have identical token paths
4. Desktop and mobile type scales must have identical token paths
5. body/md fontSize must be 16px in both breakpoints (accessibility)
6. No hardcoded values survive into generated component code
7. Every component fill in Figma is bound to a variable — never hardcoded
